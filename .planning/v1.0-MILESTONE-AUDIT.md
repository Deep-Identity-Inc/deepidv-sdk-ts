### Documentation-Level Issues (3)

**FIND-1 (LOW): `upload:error` event in example comment**
- `examples/node-basic/index.ts` line 56 references `upload:error` in a comment, but `SDKEventMap` only defines `upload:start` and `upload:complete`
- Impact: Misleading documentation; TypeScript would reject `client.on('upload:error', ...)` at compile time
- Affected: PUB-04

**FIND-2 (LOW): `page.total` accessed without null-check**
- `examples/node-basic/index.ts` line 140 uses `page.total` which is `number | undefined`
- Impact: Logs "undefined" if API omits total — cosmetic, not a crash
- Affected: PUB-04

**FIND-3 (MEDIUM): Batch presign sends single contentType**
- `packages/core/src/uploader.ts` line 359 sends `contentTypes[0]` for all files in a batch
- Impact: S3 PUT uses correct per-file Content-Type, so uploads succeed. Presign metadata is wrong for mixed-format batches (e.g., JPEG doc + PNG selfie)
- Risk: Low in practice (most identity verifications use same format). Architecturally unsound for future multi-format support
- Affected: UPL-01, UPL-04
