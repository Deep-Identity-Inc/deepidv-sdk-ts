**src/auth.ts** — Two pure functions. Per D-15: simple string concat with normalization.

```typescript
export function buildHeaders(apiKey: string, body?: unknown): Record<string, string> {
  const headers: Record<string, string> = {
    'x-api-key': apiKey,
    'Accept': 'application/json',
  };
  if (body !== undefined) {
    headers['Content-Type'] = 'application/json';
  }
  return headers;
}

export function buildUrl(base: string, path: string): string {
  const cleanBase = base.replace(/\/$/, '');
  const cleanPath = path.startsWith('/') ? path : `/${path}`;
  return `${cleanBase}${cleanPath}`;
}
```

**src/retry.ts** — Retry wrapper. Per D-01: per-attempt timeout (handled by caller). Per D-02: Retry-After cap at 60s. Per D-03: defaults from config. Per D-04: emit retry event BEFORE sleeping.

Key implementation:
- `withRetry<T>(fn: () => Promise<T>, config: { maxRetries: number; initialDelayMs: number }, emitter: TypedEmitter): Promise<T>`
- Loop from attempt=0 to maxRetries. On retryable error, emit 'retry' event with `{ attempt: attempt + 1, delayMs, error }`, then sleep.
- `isRetryable(err)`: true for status 429, 500-599, NetworkError, TimeoutError. False for all other status codes.
- `computeDelay(err, attempt, initialDelayMs)`: check Retry-After header first (from err.response.headers), cap at 60000ms. Otherwise: `Math.floor(Math.random() * Math.min(initialDelayMs * Math.pow(2, attempt), 30000))`.
- `extractRetryAfter(err)`: if err has .response?.headers?.['retry-after'], try parseInt. If NaN, try Date.parse and compute seconds from now. Return null if invalid.

Constructor takes `ResolvedConfig` and a `TypedEmitter`.

**request<T>(method: string, path: string, options?: { body?: unknown; timeout?: number }): Promise<T>**

The method wraps a single-attempt fetch call inside `withRetry()`. Each attempt:

1. Build URL via `buildUrl(config.baseUrl, path)` (D-15)
2. Create a NEW `AbortController` (critical — never reuse)
3. Set timeout via `setTimeout(() => controller.abort(), options?.timeout ?? config.timeout)` (D-01: per-attempt)
4. Emit 'request' event: `{ method, url }`
5. Call `(config.fetch)(url, { method, headers: buildHeaders(config.apiKey, options?.body), body: options?.body ? JSON.stringify(options.body) : undefined, signal: controller.signal })`
6. Clear timeout
7. If response ok (2xx): parse JSON, emit 'response' event with `{ status, url, durationMs }`, return parsed body as T
8. If not ok: read response body, build `RawResponse` object `{ status, headers: Object.fromEntries(response.headers.entries()), body }`, then:
   - 401 -> throw `new AuthenticationError(message, config.apiKey, { response, cause })`
   - 429 -> extract Retry-After, throw `new RateLimitError(message, { retryAfter, response, cause })`
   - 400 -> throw `new ValidationError(message, { status: 400, response, cause })`
   - Other 4xx/5xx -> throw `new DeepIDVError(message, { status, code: 'api_error', response })`
9. Catch block: if `AbortError` -> throw `new TimeoutError(...)`. Else -> throw `new NetworkError(..., { cause: err })`.

The `withRetry` wrapper handles the retry loop. On final failure, emit 'error' event.

Also provide convenience methods:
- `get<T>(path: string, options?): Promise<T>` -> `this.request('GET', path, options)`
- `post<T>(path: string, body: unknown, options?): Promise<T>` -> `this.request('POST', path, { ...options, body })`
- `put<T>(path: string, body: unknown, options?): Promise<T>` -> `this.request('PUT', path, { ...options, body })`
- `patch<T>(path: string, body: unknown, options?): Promise<T>` -> `this.request('PATCH', path, { ...options, body })`
- `delete<T>(path: string, options?): Promise<T>` -> `this.request('DELETE', path, options)`

Write tests FIRST in `src/__tests__/client.test.ts`. Use msw `setupServer` from the test setup file. Use `http.post()`, `http.get()` handlers from msw to mock API responses. Test each error mapping (401 -> AuthenticationError, 429 -> RateLimitError, etc.) and the happy path.
