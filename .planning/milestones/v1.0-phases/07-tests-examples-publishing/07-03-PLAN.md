Create `.github/workflows/ci.yml` with the following exact content (per D-07, D-08):

```yaml
name: CI

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

jobs:
  test:
    name: Test (Node ${{ matrix.node-version }})
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [18, 22]
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: 'pnpm'

      - run: pnpm install --frozen-lockfile
      - run: pnpm build
      - run: pnpm test
      - run: pnpm lint
      - run: pnpm format:check

      - name: Consumer type check
        run: cd test/consumer && npx tsc --noEmit
```

Key details:
- `pnpm/action-setup@v4` (NOT v3 — v3 does not support pnpm v10 per RESEARCH.md Pitfall 3)
- `version: 10` matches the project's `packageManager: pnpm@10.28.0`
- `--frozen-lockfile` prevents lockfile mutations in CI
- `pnpm build` runs before tests (required for consumer type check and for msw handler setup)
- Node matrix: 18 + 22 per D-07 (min supported + current LTS)
- Lint and format:check included (Claude's discretion — natural CI addition)
- Consumer type check runs last (depends on build output)

Create `.github/workflows/publish.yml` with the following exact content (per D-06, D-08):

```yaml
name: Publish

on:
  push:
    branches: [main]

concurrency: ${{ github.workflow }}-${{ github.ref }}

permissions:
  contents: write
  pull-requests: write
  id-token: write

jobs:
  publish:
    name: Publish to npm
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'pnpm'
          registry-url: 'https://registry.npmjs.org'

      - run: pnpm install --frozen-lockfile
      - run: pnpm build

      - name: Create Release Pull Request or Publish
        uses: changesets/action@v1
        with:
          publish: pnpm publish -r --no-git-checks --provenance
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
```

Key details:
- `pnpm/action-setup@v4` with `version: 10` (matches project)
- `actions/setup-node@v4` with `registry-url` (required for npm auth)
- `concurrency` prevents parallel publish runs
- `permissions: id-token: write` enables npm provenance attestation (Claude's discretion — strongly recommended for public SDKs)
- `permissions: contents: write` + `pull-requests: write` allows changesets/action to create Version PR
- `--no-git-checks` required in CI (per RESEARCH.md Pitfall 2 — changesets/action runs in detached HEAD)
- `--provenance` adds npm provenance linking package to GitHub Actions run
- `changesets/action@v1` handles both modes: creating "Version Packages" PR when changesets exist, and running publish when version PR is merged
- Uses `secrets.NPM_TOKEN` (must be set manually as GitHub repo secret — documented in user_setup)
- Node 22 for publish (current LTS, not the matrix — only need one node version to publish)
